# AI面试复盘助手 - 项目规范

## 1. 项目概述
- **项目名称**: AI Interview Replay Assistant (AI面试复盘助手)
- **项目类型**: 面向求职用户的交互式Web应用
- **核心功能**: 帮助用户复盘面试表现，提供专业、精准的回答质量分析和改进建议
- **目标用户**: 求职者、准备面试的用户

## 2. 功能模块

### 2.1 录入模块 (Data Input)

#### 文本模式
- **目标岗位输入**: 单行文本输入框
- **面试题目输入**: 多行文本输入框
- **回答内容输入**: 多行文本输入框（支持详细回答）
- **提交分析按钮**: 触发AI分析流程

#### 多媒体模式
- **文件上传**: 支持音频文件（mp3, wav, m4a）和视频文件（mp4, mov, webm）
- **上传预览**: 显示已上传文件信息
- **处理状态**: 显示AI处理进度

### 2.2 智能诊断模块 (Analysis)

系统根据输入数据，从以下5个维度进行深度解构：

#### 1. 切题度自检
- 判定回答是否答偏
- 检测是否落入面试官的潜在陷阱
- 识别答非所问的情况

#### 2. 逻辑结构分析
- 评估是否符合 STAR 法则（情境、任务、行动、结果）
- 评估是否符合 PREP 结构（观点、理由、证据、总结）
- 检测回答的逻辑连贯性

#### 3. 案例说服力
- 检测回答中是否有具体行动动作
- 评估量化结果（数字、百分比、具体数据）
- 分析可迁移技能展示

#### 4. 表达质量评估
- 自动统计废话率（呃、那个、然后等）
- 语速平稳度评估
- 语言流畅度分析

#### 5. 满意度预估
- 模拟面试官心理
- 给出综合满意度百分比
- 分项评分（清晰度、说服力、深度）

### 2.3 结果输出模块 (Output)

#### 结果一：核心问题清单
- 总结回答中存在的致命伤
- 问题示例：
  - 缺乏结论先行
  - 案例无重点
  - 逻辑跳跃
  - 缺乏数据支撑
  - 废话率过高

#### 结果二：黄金重写建议
- 在保留用户真实案例基础上
- 输出一份逻辑完美、措辞专业的"参考答案"
- 遵循STAR法则组织
- 提供专业的表达方式

#### 结果三：预判式追问
- 列出3个面试官基于当前回答极大概率会继续深挖的问题
- 问题与当前回答紧密相关
- 覆盖不同深挖角度

## 3. 技术方案

### 前端架构
- **技术栈**: 纯HTML5 + CSS3 + JavaScript (无框架依赖)
- **AI集成**: 支持模拟分析和OpenAI兼容API
- **文件处理**: FileReader API用于本地文件预览
- **数据存储**: localStorage用于配置持久化

### 分析引擎
- **模拟模式**: 基于规则的智能分析（免费使用）
- **API模式**: 调用真实AI API进行深度分析

### 分析评分体系
- 总分：100分
- 各维度权重：
  - 切题度：25%
  - 逻辑结构：25%
  - 案例说服力：25%
  - 表达质量：15%
  - 综合满意度：10%

## 4. UI/UX设计

### 页面布局
- 顶部: Logo和标题
- 中部: 模式切换标签 + 输入/上传区域
- 结果展示: 分维度诊断卡片 + 问题清单 + 重写建议 + 追问预测

### 配色方案
- **主色**: #4F46E5 (靛蓝色 - 专业可信)
- **辅助色**: #10B981 (绿色 - 正面评价)
- **警示色**: #EF4444 (红色 - 问题指出)
- **警告色**: #F59E0B (橙色 - 改进建议)
- **背景色**: #F9FAFB (浅灰白)
- **文字色**: #1F2937 (深灰)

### 诊断结果展示
- 五维雷达图/分项评分条
- 问题清单：红色致命伤 + 橙色改进点
- 重写建议：可折叠的参考答案
- 追问预测：模拟面试官口吻
